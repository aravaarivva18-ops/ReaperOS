# ADR-004: Автоматизированная фоновая очистка системы Reaper OS

* **Статус**: Принят (Accepted)
* **Дата**: 2026-05-26

## Контекст
В процессе эксплуатации Reaper OS на хост-системе (macOS) накапливается значительный объем временных файлов и кэш-директорий (кэш Google Chrome, кэш Homebrew, временные логи и файлы чатов .gemini/tmp, кэши Playwright и Swift/node-gyp). В отсутствие регулярной очистки это приводит к деградации свободного места (потери более 20–30 GB) и засорению контекста агентов избыточными файлами/скиллами.

Нам необходимо архитектурное решение для регулярной, безопасной и наблюдаемой очистки неиспользуемых временных ресурсов без вмешательства пользователя.

## Решение
Принято решение реализовать встроенный механизм очистки:
1. **Инструмент очистки (`tools/system_cleanup.py`)**: Python-скрипт с защитой от удаления критических данных. Скрипт:
   - Проверяет существование и размеры целевых кэш-директорий.
   - Выполняет очистку только при превышении заданных порогов или по прямым указаниям.
   - Поддерживает безопасный запуск `--dry-run` для анализа планируемой очистки без удаления.
   - Записывает логи операций во внешний файл логов.
2. **Фоновый запуск**: Настройка еженедельного запуска скрипта через планировщик `cron` (каждое воскресенье в 03:00).
3. **Наблюдаемость (Telemetry/Observability)**: Запись всех этапов работы в `logs/cleanup.log` с указанием точного объема освобожденного места.

## Последствия
* **Автоматизация**: Освобождение диска от мусора происходит в фоновом режиме без необходимости ручных команд.
* **Безопасность**: Защитные проверки в Python предотвращают удаление важных пользовательских данных.
* **Наблюдаемость**: Полный лог работы сохраняется в проекте, облегчая аудит.
