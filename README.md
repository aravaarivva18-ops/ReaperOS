# ReaperOS v12.2 (Frontier-Y) — Инструкция по установке и настройке

**ReaperOS** — это резидентная агентная среда и фреймворк самоисцеляющейся памяти проекта (Self-Healing Project Memory) на базе протокола **Trinity** (Conductor + Reaper Engine + Ruflo) с локальной LLM/Embeddings обработкой.

---

## 📋 Системные требования

*   **ОС**: macOS (рекомендуется Apple Silicon M1/M2/M3 для работы MLX).
*   **Окружение**: Python 3.12 (строго).
*   **Зависимости**: компилятор C++, Git, Homebrew (рекомендуется).

---

## 🚀 Быстрый старт (Пошаговое руководство)

### Шаг 1. Клонирование репозитория
Склонируйте репозиторий с вашего GitHub на новый MacBook:
```bash
git clone <ссылка_на_ваш_репозиторий>
cd ReaperOS
```

### Шаг 2. Создание и активация виртуального окружения
Создайте изолированную среду Python 3.12 (она должна называться строго `.reaper_venv`, чтобы игнорироваться Git):
```bash
python3.12 -m venv .reaper_venv
source .reaper_venv/bin/activate
```

### Шаг 3. Установка зависимостей проекта
Установите все необходимые библиотеки и зарегистрируйте скрипты запуска через `pyproject.toml` в режиме разработки:
```bash
pip install --upgrade pip setuptools wheel
pip install -e .
```

### Шаг 4. Загрузка локальных ИИ-моделей (Весов)
Для локального инференса без внешних API-ключей запустите скрипт автоматической загрузки. Он скачает квантованную модель `Qwen2.5-0.5B` для генерации и `all-MiniLM-L6-v2` для векторного поиска в локальную папку `local_models/weights/`:
```bash
python3 tools/download_weights.py
```
*Примечание: Загрузка занимает ~1 ГБ дискового пространства.*

## ⚡ Управление процессами (Trinity Daemon)

Запуск и остановка фоновых процессов ИИ-агентов выполняются через bash-скрипты.

### Запуск ядра Trinity:
```bash
./trinity-start.sh
```
Скрипт активирует виртуальное окружение, запускает демон-обработчик и планировщик в фоновом режиме.
*   **Лог демона**: `~/.reaper_daemon.log`
*   **Лог планировщика**: `~/.reaper_brain.log`

### Безопасная остановка ядра:
```bash
./trinity-down.sh
```
Скрипт корректно завершает процессы `engine.daemon` и `engine.main`, удаляет временные сокет-файлы и делает автоматический Git-коммит (Snapshot) текущего состояния репозитория.

## 🔮 Интеграция с Obsidian (Визуальный интерфейс)

ReaperOS сохраняет хроники, архитектурные решения и карты памяти в стандартных файлах Markdown (`.md`).
1.  Установите приложение **Obsidian** на Mac.
2.  Выберите опцию **"Open folder as vault"** (Открыть папку как хранилище).
3.  Укажите путь к папке репозитория `ReaperOS`.
4.  Вам станут доступны:
    *   `REAPER_MEMORY.md` — семантическая база памяти проекта.
    *   `GLOBAL_CHRONICLE.md` — долгосрочный лог ключевых сессий.
    *   `Dashboard.md` — сводный пульс системы.

## 📈 Визуальный веб-дашборд

Вы также можете запустить красивый Bento Grid OLED-дашборд для мониторинга в браузере:
```bash
cd dashboard
python3 -m http.server 8080
```
После запуска откройте в браузере: **`http://localhost:8080`**. Вы сможете наблюдать реалтайм-логи, загрузку токенов, задержку API и карту связей MemPalace.
